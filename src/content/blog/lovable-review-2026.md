## Lovable's Weaknesses

### ❌ Credit System Is Punishing

The credit system is the #1 complaint. Complex prompts cost more credits, and Lovable isn't always transparent about how many it will use. We had a single feature request consume 8 credits unexpectedly.

**Comparison:**
- Bolt.new: Token-based (more predictable, more transparent)
- v0.dev: Component credit system (free tier is generous)
- Lovable: Message credits (can feel arbitrary)

### ❌ Mobile UI Needs Work

Lovable's generated apps look great on desktop. Mobile responsiveness is inconsistent — you often need 10-20 follow-up messages just to fix spacing and layout on small screens.

### ❌ Custom Domains = Supabase Lock-In (Partly)

To get custom domain + SSL, you need Supabase's paid tier ($25/mo). If you're already paying $50/mo for Lovable, your real cost is $75/mo minimum for a "real" deployment. This isn't always obvious up front.

Yes. You can sync to GitHub and own your code completely. No vendor lock-in on the code itself (though Supabase integration creates some database lock-in).
